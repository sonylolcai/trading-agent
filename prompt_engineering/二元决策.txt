# 交易二元决策树

本文件按二元判断方式组织交易决策逻辑。每一步只问一个问题：是 / 否。如果答案不清楚，默认进入“等待”，而不是强行做多或做空。

核心顺序：

1. 先判断市场状态。
2. 再判断方向偏好。
3. 再找入场信号。
4. 再检查风险收益（含止损、目标）。
5. **交易者方程（10.3）在已确定止损与目标之后、下单（§11）之前做最终裁定。**
6. ~~入场后逐K管理。~~（PA Agent 不持仓，跳过 §12–§13）
7. ~~市场状态变化时立刻切换策略。~~（单次分析；请用户重新跑分析）

**阶段划分（与 PA Agent 两阶段分析一致）：**
- 阶段一（诊断闸门 §0–§2）：只做定性判断，**不做**交易者方程数值检验。
- 阶段二（策略与执行 §3–§11、§14）：结构分支 → 入场信号(§9) → 风险收益(§10，含 10.3) → 下单方式(§11)。

---

## 0. 总原则

### 0.1 是否看得懂当前市场？

是：继续下一步。

否：等待。

处理：
- 不开仓。
- 不猜方向。
- 不因为“感觉要动了”而交易。
- 等待更多K线，让结构变清楚。

### 0.2 是否具备继续深入分析的条件？（定性，非交易者方程）

是：允许进入阶段二，继续评估具体策略分支、入场与风控。

否：等待。

本步仅做**定性**判断（结构是否足以支持后续分析），须考虑：
- 当前周期位置是否可识别。
- 长程结构窗口背景是否可描述。
- 价格位置是否极端模糊。

**不在此步**用胜率×回报做数值检验；数值检验见 **§10.3**（止损与目标确定之后）。

### 0.3 交易者方程（原则提醒，非执行节点）

公式：胜率 × 预期回报 > 败率 × 预期风险

本条为总原则：**最终是否允许下单，以 §10.3 的评估为准。**

- 阶段一（诊断）**不要**填写本节点为「是/否」闸门。
- 阶段二在 **§10.1–§10.2 已定止损与目标之后** 必须执行 **§10.3**，通过后才可进入 **§11 下单方式**。

## 1. 是否可以决策

### 1.1 数据是否足够？

否：
- K线数量少于20根。
- 数据质量不足。
- 图形太乱无法判断。

处理：
- 输出：无法决定。
- 操作：等待。
- 不加载交易策略。
- 不做任何方向性判断。

是：进入下一步。

### 1.2 是否能识别市场周期？

必须能归入以下状态之一：

- 尖峰 / 极速上涨或极速下跌。
- 微型通道。
- 紧凑通道。
- 常规通道。
- 宽通道 / 台阶。
- 趋势型交易区间。
- 普通交易区间。
- 极端交易区间。

否：
- 输出：unknown。
- 操作：等待。
- 原因：诊断不清，决策必错。

是：进入下一步。

### 1.3 当前市场是否极端混乱？

判断条件：
- 大量K线重叠。
- 上下影线频繁。
- 20EMA水平并与价格纠缠。
- 没有清晰方向。
- 任意入场接近抛硬币。
- 胜率接近50%，扣除滑点手续费后期望为负。

是：
- 状态：极端交易区间。
- 操作：不交易。
- 等待以下任一信号：
  - 连续3到5根强趋势棒突破。
  - 价格形成清晰通道。
  - 重要消息后市场选择方向。

否：进入方向判断。

## 2. 多空方向判断

### 2.1 是否存在明确惯性方向？

判断依据：
- 20EMA是否上倾或下倾。
- 最近20根K线重心是否上移或下移。
- 波段高低点是否有序。
- K线重叠度是否低。
- 是否连续出现同方向趋势棒。
- 价格是否持续运行在EMA一侧。

否：
- 进入区间逻辑。
- 不做趋势跟踪。
- 只考虑区间边界交易，或等待。

是：进入趋势 / 通道 / 尖峰逻辑。

### 2.2 长程大背景方向与近期方向的关系？

提取方法：
- 长程大背景方向：在 K100-K1 的完整结构上判断主要趋势方向
  （多头/空头/区间/中性）
- 近期方向：来自 §2.3（当前方向判断）的结果
- 比较两者关系，分三种情况：

**同向（一致）：**
- 强共振，置信度可提高。
- 顺势交易的可靠性最高。

**反向（冲突）：**
- **不否决近期方向。** Al Brooks："新趋势比旧有趋势更能代表当前的现状，
  也应该成为你交易的核心"（《逐棒分析》第88页）
- 近期方向是当前交易的主方向，大背景方向作为风险提示。
- 策略调整：
  - 不输出任何仓位/资金/手数建议；仅用“信号强弱/风险等级/是否建议交易”表达保守程度。
  - htf_context 中记录大背景方向（如"AIS"），作为磁力位警告。
  - 若近期方向明确（§2.3 非中性）且 §2.5 惯性够强 →
    按近期方向执行，不等待。
  - 若近期信号弱（§2.3 中性 + §2.4 否 + §2.5 否）→ 等待更多K线确认。
  - 不要求"必须出现楔形反转/双重顶底等特殊形态"才允许交易——
    近期尖峰本身就是有效的顺势方向（"新趋势是交易核心"）。

**中性/无明确大背景方向：**
- 中立，按近期信号窗口独立判断。
- 交易决策完全由 §2.3–§2.5 决定。

### 2.3 当前方向是多头还是空头？

多头条件：
- EMA上倾。
- HH + HL结构。
- 收盘重心上移。
- 多头趋势棒占优。
- 回撤小且被买入。

空头条件：
- EMA下倾。
- LL + LH结构。
- 收盘重心下移。
- 空头趋势棒占优。
- 反弹小且被卖出。

如果都不明显：
- 视为中性。
- 进入区间 / 等待逻辑。

### 2.4 当前是否处于 Always In 状态？

Always In Long（AIL）条件：
- 最近20根K线多数收在EMA上方。
- EMA上倾，且价格回撤浅。
- HH + HL结构清晰。
- 多头趋势棒占优，空头反击缺少跟随。

⚠️ 20GB特殊规则：连续约20根K线未触及EMA（即"20GB"，定义见「文件16-K线信号识别.txt」），表示趋势极强。在20GB状态下：
  - 绝对不逆势。
  - 首次触及EMA是顺势刮头皮机会（非反转信号）。
  - 两次尝试失败（触及EMA后无顺势跟随）则放弃，不做第三次。
  - 识别到20GB时，detected_patterns 写入 `20gb`。

Always In Short（AIS）条件：
- 最近20根K线多数收在EMA下方。
- EMA下倾，且反弹弱。
- LL + LH结构清晰。
- 空头趋势棒占优，多头反击缺少跟随。

⚠️ 20GB空头对称规则：连续约20根K线未触及EMA，首次触及EMA是顺势做空机会。

是：
- AIL：默认只找做多或持有多头，不做首个做空反转信号。
- AIS：默认只找做空或持有空头，不做首个做多反转信号。
- 只有满足趋势线突破 + 前极点测试失败，才允许进入反转逻辑。

否：
- 视为中性或弱惯性。
- 降低方向置信度，优先等待区间边界、二次入场或更多K线确认。

### 2.5 当前惯性强度是否足够支持趋势跟踪？

强惯性条件：
- K线重叠少。
- 回撤通常不超过1到3根K线。
- 趋势方向K线连续出现，收盘接近方向极点。
- 没有连续两根K线收在EMA另一侧。
- 反向信号缺少跟随。

是：
- 允许趋势跟踪或顺势回撤交易。
- 逆势信号默认视为回撤或旗形，而不是反转。

否：
- **不代表 gate=wait，不在此节点短路阶段二。** 继续进入策略分支（§3–§6），切换为"等待反弹到位后的顺势信号"模式。
- 具体处理：
  - 不做激进趋势跟踪（不追突破、不在反弹中追顺势）。
  - 转为等待反弹衰竭后的入场信号：宽通道等反弹至上边界/EMA附近出现空头信号棒；常规通道等回撤至38%–50%出现顺势信号棒。
  - 在 watch_points 中明确写出触发条件（位置+形态），而不是泛泛说"等待结构明朗"。
- **只有当市场状态完全无法识别（§1.2 unknown）或极端混乱（§1.3 extreme_tr）时，才在阶段一输出 gate=wait。惯性暂时中断不构成 gate=wait 的条件。**

## 3. 是否为尖峰 / 极速行情

### 3.1 是否满足尖峰条件？

⚠️ 以下分级定义与「市场诊断框架.txt」状态1中【统一 Spike 分级定义】完全一致，以该定义为准。

做多尖峰条件分级：
- 弱尖峰候选（1根）：恰好1根超大突破棒 + 下一根有跟随 → 只记录 `spike_candidate`，**1根时 cycle_position 不出 spike**，输出 trending 或 wait，watch_points 写明"等第二棒确认，若第二棒为强多头趋势棒且重叠<30%则切换spike路由"；禁止追单。
- 最低可路由尖峰（2根）：2根以上连续大型多头趋势棒，实体重叠 < 30%，收盘持续创新高，突破关键阻力或区间上边界，有紧迫感。**2根可确认spike路由，但仅允许SPS等待回撤，禁止SCS追单；3根以上才允许SCS追单。** 2根spike的策略：export spike路由 → spike_stage=active → 可用操作仅限于SPS（等回撤后入场），不允许SCS追单。若价格直接继续推进到3根以上，自动升级为允许SCS。
- 标准尖峰（3~5根）：3到5根，实体重叠 < 20%，几乎无上尾线。SCS/SPS/FFES 均可使用（参见文件16 尖峰态专属信号规则）。
- 强尖峰/高潮风险（6根以上，或任意序列+衰竭信号）：按高潮判断（见市场诊断框架状态1），**禁追**，优先等待结构恢复或回撤后再评估

做空尖峰条件分级（对称）：
- 弱尖峰候选：1根超大突破棒 + 下一根有跟随 → 同上，禁止追单
- 最低可路由：2根以上，实体重叠 < 30%，收盘持续创新低，跌破关键支撑
- 标准尖峰：3~5根，实体重叠 < 20%，几乎无下尾线
- 强尖峰/高潮风险：同多头对称处理

否：进入通道或区间判断。

是：继续。

### 3.2 尖峰是否仍在进行中？

是：
- 只顺势，不反转。
- 上涨尖峰：只做多。
- 下跌尖峰：只做空。
- 禁止因为“涨太多 / 跌太多”逆势。

否：
- 进入尖峰结束后的三路径判断。

### 3.3 尖峰进行中是否允许追？

必须全部满足：
- 连续3根以上强趋势棒。
- 突破重要位置。
- 长程结构窗口同向。
- 没有高潮衰竭信号。
- 默认结构止损为最近回撤/趋势棒极点外1跳；若结构目标超过1.5R，只能按 §10.3 的尖峰 widen-stop 规则放宽，最终止损不得超过尖峰高度50%。
- 不输出任何仓位比例；尖峰追单天然风险更高：trade_confidence 必须下调，目标更保守，不在高潮/衰竭信号后追单。

是：
- 使用 SCS 追尖峰信号。
- 可用市价单或止损单入场。
- 止损：最近趋势棒或回撤极点外1跳动。
- 目标：尖峰高度的等量移动。

否：
- 不追。
- 等待回撤入场。

### 3.4 尖峰是否出现第一次暂停或回撤？

否：
- 继续观察。
- 不逆势。
- 不新增反向交易计划。

是：
- 如果回撤只有2到4根K线，且出现顺势信号棒：
  - 使用 SPS 尖峰回撤入场。
  - 顺势入场。
- 如果形成第一次旗形：
  - 使用 FFES 最终旗形入场。
  - 等旗形突破后顺势入场。

### 3.5 尖峰后进入哪条路径？

#### 路径A：回撤小于38.2%到50%

判断：
- 趋势大概率恢复为通道。
- 概率约60%。

处理：
- 从尖峰策略切换为通道策略。
- 不再盲目追。
- 等回撤到EMA、趋势线、通道边界附近顺势入场。
- 不输出任何仓位建议；仅在结构清晰且交易者方程通过时给出三价方案。

#### 路径B：回撤超过61.8%，并开始横盘

判断：
- 尖峰失败，进入交易区间。
- 概率约30%。

处理：
- 停止趋势跟踪。
- 改为区间高卖低买。
- 不追突破。
- 中间位置不交易。

#### 路径C：趋势线突破 + 前极点测试失败

判断：
- 可能反转。
- 概率约10%，但需要双确认。

处理：
- 不在第一根反转K线入场。
- 等二次入场。
- 不输出任何仓位建议；反转计划需更保守的目标与更严格的确认。

### 3.6 是否出现高潮？

⚠️ 高潮判定统一规则（见市场诊断框架状态1）：任意连续强趋势棒序列 + 出现以下任一衰竭信号即触发：
- 长尾线（尾线 > 实体长度50%）
- 小实体棒（实体 < 近期均值30%）
- 反向趋势棒收盘

买进高潮触发（不依赖固定根数）：
- 连续上涨趋势棒序列 + 出现上述任一衰竭信号
- 特别警示：6根以上连续多头趋势棒时，即使尚未出现衰竭，也进入高潮预警状态

卖出高潮触发（对称）：
- 连续下跌趋势棒序列 + 出现下述任一衰竭信号（长下影、小实体、多头趋势棒）
- 6根以上连续空头趋势棒时进入高潮预警

是：
- 已有顺势仓位：本 Agent 不做持仓后管理，不讨论减仓/加仓/移止损；仅提示“高潮后禁追、优先等待结构清晰”。
- 禁止继续追原方向。
- 等至少10根K线或二次入场。

否：继续按尖峰或通道策略处理。

### 3.7 是否为第二次或第三次高潮？

第二次高潮：
- 不追。
- 开始保护利润。
- 等回撤后再考虑新顺势交易。

第三次高潮：
- 高概率接近趋势末端。
- 本 Agent 不做持仓后管理，不输出平仓/减仓/移止损等指令。
- 不追原方向。
- 只在反转确认后考虑反向。

## 4. 是否为通道

### 4.1 是否出现有序波段结构？

上涨通道：
- 高点逐步抬高：HH。
- 低点逐步抬高：HL。
- 至少两组HH + HL。

下跌通道：
- 低点逐步降低：LL。
- 高点逐步降低：LH。
- 至少两组LL + LH。

否：
- 进入区间判断。

是：
- 继续。

### 4.2 通道方向是上涨还是下跌？

上涨通道：
- 核心认知：上涨通道 = 空头旗形。
- 顺势做多仍优先。
- 但通道越久，越要警惕向下突破。

下跌通道：
- 核心认知：下跌通道 = 多头旗形。
- 顺势做空仍优先。
- 但通道越久，越要警惕向上突破。

## 5. 通道宽度判断

### 5.1 是否为微型通道？

条件：
- 2到10根K线。
- 几乎没有回撤。
- 每根K线高低点连续同向推进。
- 常是尖峰的一部分。

是：
- 不逆势。
- 不追第一个突破。
- 第一个突破尝试通常失败。
- 等突破失败后顺势入场。

处理：
- 上涨微型通道：向下突破失败后做多。
- 下跌微型通道：向上突破失败后做空。
- 止损：突破极点外1跳动。
- 目标：微型通道高度或尖峰测量移动。

否：继续。

### 5.2 是否为紧凑 / 窄通道？

⚠️ 通道分类统一以回撤幅度为主维度，斜率、绝对跳动数和视觉宽窄只作辅助描述，避免双维度判断结果不一致。

条件（全部满足）：
- 回撤 < 30%（量化，含1-3跳动微回撤）。
- K线重叠少（平均重叠 < 20%）。
- 趋势棒比例 > 65%。
- EMA紧贴价格，两者间无连续2根K线完全在EMA另一侧。
- （辅助描述）斜率大、几乎垂直于水平基线。

是：
- 只做顺势。
- 不做逆势。
- 不追通道边界突破。
- 等回撤到EMA或趋势线附近入场。

上涨窄通道：
- 首选：High1 / High2 回撤买入。
- 止损：信号棒低点外1跳动。
- 目标：最近结构位、通道上轨或测量移动候选；最终回报÷风险不得超过1.5。

下跌窄通道：
- 首选：Low1 / Low2 反弹卖出。
- 止损：信号棒高点外1跳动。
- 目标：最近结构位、通道下轨或测量移动候选；最终回报÷风险不得超过1.5。

否：继续。

### 5.3 是否为常规通道？

条件：
- 回撤30%到50%（量化）。
- 波段清晰，每波段至少有2-3根信号K线。
- （辅助描述）通道斜率适中。
- 每个波段有交易空间。

是：
- 主要顺势。
- 允许少量逆势刮头皮，但必须在通道边界。
- 逆势计划门槛更高：必须在边界出现强信号，且结构止损/目标清晰；否则不做逆势。

处理：
- 顺势入场：回撤到38.2%到50%区域，出现信号棒。
- 逆势入场：必须到达通道边界 + 明确反转K线。
- 止损：信号棒极点外1到2跳动。
- 顺势目标：通道对边或测量移动候选，必须截断在1.5R内最近结构位。
- 逆势目标：通道中线，到了就走；不得超过1.5R。

否：继续。

### 5.4 是否为宽通道 / 台阶？

条件（全部满足）：
- 回撤深，50%到78.6%（量化，主维度）。
- 波段大，每波段通常超过10根K线。
- K线重叠 > 40%。
- 突破后常有突破测试。
- （辅助描述）斜率小（< 30°）。

⚠️ 与 trending_tr 边界模糊时：若已有3组有效 HH+HL / LL+LH 且可画稳定通道线，输出 broad_channel；若只有2组或通道线不稳定，输出 trending_tr。不得仅因回撤45%-55%或均线微倾斜就把2组结构升级为 broad_channel。

是：
- 可以双向交易。
- 顺势优先，但不执着。
- 核心模式：突破后不追，等突破测试。

上涨宽通道：
- 下边界做多。
- 上边界可评估小目标的逆势刮头皮（仅当强信号 + 结构止损清晰）。
- 上破通道线不追，等回踩。

下跌宽通道：
- 上边界做空。
- 下边界可评估小目标的逆势刮头皮（仅当强信号 + 结构止损清晰）。
- 下破通道线不追，等反弹测试。

风控：
- 止损：信号棒极点外1跳；边界噪声大时扩至最近波段极点外1跳。
- 目标：通道中线、对侧边界、或下一台阶候选；最终取不超过1.5R的最近结构目标。

否：继续评估是否为区间。

### 5.5 是否为收缩台阶？

条件：
- 至少三段推进。
- 每段推进幅度递减。
- 动能衰退。
- 趋势线多次重画并变平。

是：
- 顺势追单风险上升：trade_confidence 下调，避免在最后一推追顺势；优先等突破后的回撤/二次入场确认。
- 不在最后一推追顺势。
- 准备反转或区间策略。
- 等趋势线突破 + 极点测试失败后再反向交易。

否：
- 继续按当前通道类型执行。

## 6. 是否为交易区间

### 6.1 是否存在清晰上下边界？

⚠️ 先确认已通过「市场诊断框架.txt」中的【统一判定树】走到区间态。满足以下全部条件才进入区间策略：
- 无满足3组的 HH+HL 或 LL+LH 序列（否则走通道策略）
- 上下边界至少各测试2次
- EMA 水平或斜率 ≤ 10°
- 大量K线重叠

否：
- 重新评估是否为 trending_tr 或 broad_channel。

是：
- 进入区间策略。

### 6.2 是普通交易区间还是趋势型交易区间？

普通交易区间：
- 无明显方向偏好。
- 上下边界对称。
- 均线水平。
- 突破80%失败。

处理：
- 下边界做多。
- 上边界做空。
- 中间位置不交易。
- 不追突破。
- 目标取 min(区间宽度50%~80%, 1.5×止损距离)；若低于1R且胜率不足60%，不交易。

趋势型交易区间：
- 有轻微方向偏好。
- 低点逐渐抬高或高点逐渐降低。
- 均线略微倾斜。
- 但每个波段仍可能反转。

处理：
- 顺势方向优先。
- 逆势方向仅在边界强信号且结构止损/目标清晰时才考虑。
- 顺势目标可取区间宽度60%到80%的候选，但必须截断在1.5R内。
- 逆势目标只取30%到50%，且不得超过1.5R。

### 6.3 当前价格是否在区间边界？

否，在中间1/3：
- 不交易。
- 等价格到边界。

是，在下边界：
- 等看涨信号。
- 可做多。

是，在上边界：
- 等看跌信号。
- 可做空。

### 6.4 是否出现Barbwire / 极窄区间？

条件：
- 10根以上K线高度重叠。
- 区间宽度小于平均波段30%。
- EMA水平并纠缠。
- 方向完全不明。

是：
- 不在里面交易。
- 等尖峰级别突破。
- 突破后仍不追，等回踩测试成功再顺势入场。

否：
- 继续边界交易或等待突破测试。

### 6.5 区间是否出现突破？

否：
- 继续低买高卖。

是：
- 判断突破质量。

高质量真突破：
- 连续3到5根同方向强趋势棒。
- 实体大，影线短。
- 突破边界后不立刻拉回。
- EMA开始跟随方向。
- 最好有回踩测试成功。

处理：
- 不追第一根突破，除非尖峰条件完整。
- 等回踩测试。
- 测试成功后顺势入场。
- 目标：区间高度或突破前波段的测量移动。

低质量假突破：
- 只有1到2根K线。
- 影线长。
- 无跟随。
- 突破后快速回区间。
- 出现在区间中段或低流动时段。

处理：
- 反向交易。
- 回到区间内部后，向区间对边交易。
- 止损放在假突破极点外。

### 6.6 是否为三角形形态？

⚠️ 三角形已在 pattern_enum.md 中定义（ascending_triangle / descending_triangle / symmetrical_triangle / expanding_triangle），detected_patterns 必须写入对应枚举值。

三角形是交易区间中的特殊收敛形态，本质仍是区间，但有独特的交易规则。

**上升三角形**（上边界水平 + 低点渐高）：
- 向上突破 → 等待回踩测试上边界，测试成功后买入（止损：回踩低点下方1跳）
- 向下跌破低点渐高趋势线 → 可靠卖出信号，等反弹后做空（止损：反弹高点上方1跳）

**下降三角形**（下边界水平 + 高点渐低）：
- 向下突破 → 等待回踩测试下边界，测试成功后卖出（止损：回踩高点上方1跳）
- 向上突破高点渐低趋势线 → 可靠买入信号，等回踩后做多（止损：回踩低点下方1跳）

**对称三角形**（边界收敛）：
- 等待收敛到极致后尖峰突破
- 突破后等回踩测试，顺势入场
- 入场前禁止预判突破方向

**扩张三角形**（边界向外扩散）：
- **不交易**。这是最混乱的形态，边界不可靠。
- 等待形态结束、方向明确后再评估。

风控（统一）：
- 止损：信号棒极点外1跳
- 目标：突破方向测量移动（区间高度投影）
- 不输出任何仓位/资金/手数建议
- 突破不追，等回踩测试成功后再入场（与 §6.5 突破测试原则一致）

## 7. 是否出现反转

### 7.1 是否只有单根反转K线？

是：
- 不够。
- 在趋势中，单根反转K线大多只是回撤。
- 等更多确认。

否：
- 继续检查。

### 7.2 是否同时满足两大反转条件？

条件一：
- 趋势线被收盘价有效突破。

条件二：
- 前极点测试失败。
- 上涨趋势：无法再创新高。
- 下跌趋势：无法再创新低。

否：
- 不做反转。
- 视为旗形或回撤。
- 继续按原趋势处理。

是：
- 反转成立概率明显提高。
- 第一笔更保守：优先二次入场，目标与条件更严格。
- 优先等二次入场。

### 7.3 是否出现二次入场？

二次入场类型：
- MTR后第一次回撤。
- 双顶 / 双底第二次测试。
- 趋势线突破后回测。
- 楔形突破后回撤。
- 区间边界第二次测试。

否：
- 等待。
- 不追第一信号。

是：
- 可以入场。
- 止损：二次入场信号极点外1跳动。
- 目标：反转起点、区间对边、或测量移动。
- 不输出任何仓位建议；以结构止损与交易者方程裁定是否给出三价方案。

### 7.4 二次入场是否失败？

是：
- 不做第三次尝试，除非是清晰楔形三推。
- 重新评估市场结构。
- 很可能原趋势仍未结束。

否：
- 按反转交易管理。

---

## 8. 是否出现楔形 / 三推

### 8.1 是否有三推结构？

条件：
- 同方向推进三次。
- 每次推进动能递减。
- 趋势线与通道线收敛。
- 第三推明显弱于前两推。

否：
- 忽略。
- 不强行画楔形。

是：
- 判断楔形性质。

### 8.2 楔形是回撤还是反转？

楔形回撤：
- 楔形方向与主趋势相反。
- 通常10到20根K线。
- 出现在趋势中的调整阶段。

处理：
- 顺主趋势交易。
- 上涨趋势中的下降楔形：向上突破做多。
- 下跌趋势中的上升楔形：向下突破做空。
- 目标：前极点或测量移动。

楔形反转：
- 楔形方向与主趋势相同。
- 通常20根以上。
- 出现在趋势末端。
- 动能明显衰竭。

处理：
- 不在第三推中立刻反向。
- 等楔形突破。
- 最好等突破后的回撤二次入场。
- 逆势楔形反转计划门槛更高：更严格确认、更保守目标；不输出仓位建议。
- 目标：楔形起点 + 楔形高度测量移动。

### 8.3 是否为抛物线楔形？

条件：
- 价格加速推进。
- 每一推斜率更陡。
- 出现高潮感。

是：
- 已有顺势仓位：本 Agent 不做持仓后管理，不输出减仓/加仓/移止损等指令；仅提示“高潮后禁追、优先等待结构清晰”。
- 不再追趋势。
- 等深回撤和二次测试后再考虑反转。
- 第三次高潮后，坚决避免继续追原方向。

否：
- 按普通楔形回撤或楔形反转处理。

## 9. 入场信号二元检查

任何交易入场前，都必须逐项检查。

### 9.0 信号棒是否已经收盘且质量足够？

**本节点只评估「当前是否已有合格信号棒」**（实体清晰、方向一致、可挂止损于棒极点）。与计划型限价无关。

否：
- **不得**直接 terminal=wait 结束分析。
- **必须**继续评估 **§9.0P 背景驱动限价单**（见下）。
- 未收盘的K线只作为观察对象，不作为最终信号棒。

是：
- 继续 §9.1–§9.7 检查信号棒方向、长度、背景与跟随。

### 9.0P 背景驱动限价单评估（§9.0=否 时**必须**填写）

**当 §9.0=否 时，本节点为正式第二路径**——评估周期/方向/结构位是否支持计划型限价单；**不是**在 reasoning 里私下评估。

否：
- 无计划型 setup：区间/通道 **中部**（§6.3=middle）、barbwire 且无边界锚点、无法给出结构失效 stop 与有效 target、§14 触犯。
- terminal.outcome=wait，node_id=**9.0P**（或 9.0，若未写 9.0P）。

是：
- 阶段一 gate_result=proceed，cycle_position 为宽通道/常规通道/交易区间/趋势型交易区间（或尖峰回撤结构位），且价格靠近 **支撑/阻力/通道边界/EMA/顺势回撤位**。
- **依据是周期 + 方向背景 + 结构位**，不要求独立信号棒。
- `signal_bar.bar=null`，`quality=invalid/weak`，`pattern=none`；`entry_bar` 设 not_triggered/pending。
- **必须继续 §10.1→10.2→10.3→§11**，尝试输出 `order_type=限价单` 及三价；不得在 watch_points 代替 decision 三价。

**§9.0P=是 时的 K1 对照（仅计划型限价）：**
- 只检查 **entry 是否在 K1.close 的正确一侧**（做多限价 entry < K1.close；做空限价 entry > K1.close）。
- **禁止**因 K1 影线曾触及 entry 就判失效——限价单是等**未来**回撤/反弹，不是要求 K1 未路过。
- K1 已触及 **stop** 仍判方案无效。

**§10 定价顺序（计划型限价推荐）：**
1. 定 **entry**（结构位/边界/回撤位）
2. 定 **take_profit_price（TP1）**（最近有效结构目标；**须满足 RR 1.0–1.5**）
3. 定 **take_profit_price_2（TP2）**（更远结构目标：MM、通道对边远端、区间翻测等；**必填，不受 1.5R 限制**）
4. 计算 reward = |TP1 − entry|（**仅用 TP1 做 §10.3**）
5. 以 RR≤1.5 反推：`risk_cap = reward / 1.5`，候选 stop = entry ∓ risk_cap（多减空加）
6. 若 **结构失效位**比候选 stop 更远 → 用结构 stop（RR 可能 <1.5，若 <1.0 则 reject）
7. 若结构 stop 更近 → 用结构 stop（RR 可能 >1.5，合法）
8. 几何校验：做多 stop<entry<TP1<TP2；做空 TP2<TP1<entry<stop

高质量信号棒：
- 与计划方向一致。
- 实体清晰，影线较短。
- 收盘接近计划方向极点。
- 出现在正确背景中：趋势回撤末端、区间边界、突破回踩、楔形或反转双确认之后。

低质量信号棒：
- 十字星。
- 长外包棒。
- 与前几根K线大幅重叠。
- 出现在区间中部或铁丝网中。
- 过长导致止损过大。

**低质量信号棒的处理（等待触发条件）：**
当前K线（K1）为低质量信号棒（如十字星、doji、外包棒）时，不直接放弃，而是在 watch_points 中明确下一根K线（K0）的触发条件：
- 宽通道/常规通道空头背景：若 K0 为空头趋势棒且收盘低于 K1 低点 → K0 成为新的信号棒，**止损改为 K0 高点上方1跳**，以 K0 为基准重新通过 §10.1–§10.3 评估。
- 宽通道/常规通道多头背景：若 K0 为多头趋势棒且收盘高于 K1 高点 → K0 成为新的信号棒，**止损改为 K0 低点下方1跳**，以 K0 为基准重新通过 §10.1–§10.3 评估。
- 紧凑通道空头：若 K0 为空头趋势棒（Low 1 或 Low 2 形态）→ K0 高点外1跳为止损，重新评估方程。
- **关键原则：K0 触发后的止损与交易者方程，必须以 K0 的极点为基准重新计算，而不是沿用 K1 的极点。** K0 通常比 K1 更短，止损更小，方程更容易通过。
- 当前分析轮次没有 K0，故输出 wait；**应在 watch_points 中明确写出 K0 触发条件以及触发后的止损逻辑**，而不是仅说"等待"。
- 如果 K0 不满足上述条件，继续等待，不强行入场。

### 9.1 信号K线是否已经收盘？

否：
- 等待。
- 不在K线形成过程中提前进场。

是：
- 继续。

### 9.2 信号K线方向是否与计划方向一致？

做多要求：
- 阳线。
- 收盘接近高点。
- 下影线较短。
- 最好在支撑、EMA、趋势线或区间下边界。

做空要求：
- 阴线。
- 收盘接近低点。
- 上影线较短。
- 最好在阻力、EMA、趋势线或区间上边界。

否：
- 降低 trade_confidence 或放弃。

是：
- 继续。

### 9.3 信号棒是否过长？

是：
- 风险可能过大。
- 使用资金管理止损，或等更小回撤，或放弃。

否：
- 使用信号棒极点止损。

### 9.4 是否是第一次入场？

是：
- 趋势顺势交易可以接受。
- 反转交易最好等待第二次。

否，是第二次入场：
- 胜率更高。
- 优先考虑。

### 9.5 是否有跟随？

有：
- 可以持有或加大目标。

无：
- 信号可能失败。
- 降低 trade_confidence，偏向等待确认或放弃；不输出减仓/收紧止损/退出等持仓管理指令。

### 9.6 是否为二次入场？

是：
- 胜率通常高于第一次入场。
- 反转、楔形、趋势线突破回测、区间边界第二次测试，应优先等待二次入场。
- 若二次入场价格与第一次相同或略差，说明市场要求为更多确认支付成本，结构更健康。

否：
- 趋势顺势交易可以接受第一次入场。
- 逆势反转、弱背景、交易区间中部附近，不做第一次入场，继续等待。

警惕：
- 如果二次入场价格明显优于第一次，可能是陷阱或原结构误判。
- 二次入场失败后，不做第三次尝试，除非是清晰楔形三推。

### 9.7 入场棒是否强势且有跟随？

是：
- 入场棒收盘方向与计划一致。
- 收盘接近方向极点。
- 后续1到2根K线继续同方向推进或至少不明显反向。
- 可以继续进入风险收益检查。

否：
- 信号可能失败。
- 降低 trade_confidence，偏向等待确认或放弃；不输出收紧止损等持仓管理指令。
- 若突破后无跟随，优先按假突破或失败信号处理。

## 10. 风险收益二元检查

### 10.1 是否能明确止损？

否：
- 不交易。

是：
- 继续。

【⚠️ 统一止损原则（全系统唯一标准）】

止损位置永远基于结构失效位，不基于固定跳数。固定跳数（如"边界外1~3跳"）仅用作风险过滤器——判断止损距离是否过大而应放弃，而不是止损本身的设置依据。

**通用规则：止损 = 信号棒极点外1跳（做多：信号棒低点下方1跳；做空：信号棒高点上方1跳）**

各市场状态的止损位置：

| 状态 | 止损设置位置 | 过滤器（超过则考虑放弃） |
|------|------------|------------------------|
| 尖峰进行中（SCS追单） | 最近回撤K线极点外1跳；若无回撤，则最近一根趋势棒极点外1跳；若结构目标>1.5R，仅尖峰可用widen-stop | 放大后止损 > 尖峰高度50% |
| 尖峰回撤（SPS入场） | 回撤极点外1跳（做多：回撤低点下方1跳；做空：回撤高点上方1跳） | 止损 > 尖峰高度50% |
| 普通趋势/通道顺势 | 信号棒极点外1跳 | 止损 > 通道宽度40% 或 > 2倍ATR |
| 普通区间边界 | 信号棒极点外1跳，边界作为结构参考 | 止损 > 区间宽度10% |
| 宽通道（边界噪声大） | 信号棒极点外1跳；边界刺穿频繁时扩至波段极点外1跳 | 止损 > 通道宽度50% |
| 铁丝网/Barbwire | 不做，或等待突破后以突破棒极点外1跳止损 | 边界外1~2跳无效，最少需3~5跳缓冲才考虑 |
| 反转交易 | 反转信号棒极点外1跳 | 止损 > 2倍ATR |

资金管理止损（8跳动/信号棒60%高度）的角色：
- 仅作为"止损是否过大"的判断阈值
- 若图表结构止损位置不在此范围内，不得用固定跳数替代图表止损
- 若信号棒过长导致结构止损过大 → 等更小信号，或放弃交易

### 10.2 止损是否过大？

以下任一成立即视为过大（放弃交易或等更小信号）：
- 止损 > 止盈目标距离（盈亏比 < 1:1）
- 止损 > 通道宽度40%（通道态）
- 止损 > 区间宽度10%（区间态）
- 止损 > 2倍ATR
- 资金管理止损阈值（8跳动/信号棒60%高度）只是辅助参考：若结构止损大于此阈值，优先等待更小信号；不得用固定跳数替代图表结构止损

是：
- 放弃。
- 或等待更小信号。
- 不为了进场而扩大止损。

否：
- 继续。

### 10.3 交易者方程是否通过？

否：
- 不交易。

是：
- 允许下单。

- 基本要求：
- 刮头皮：至少1:1，但胜率需大于60%；若 TP1 无法达到1R，不交易。
- 波段/趋势/反转：**TP1** 以最近有结构支撑的价位为准；**盈亏比硬上限 1.5:1**（回报÷风险 ≤ 1.5，仅基于 TP1）。
- **TP2** 为更远结构目标（MM 等），必填，不参与 §10.3 与 RR 校验。
- 反转交易：TP1 保守；勿在 JSON 中编造具体手数或资金规模。
- 数值检验：胜率%×回报 > (100−胜率)%×风险（回报、风险**仅来自 TP1**）才允许下单。

**尖峰交易与1.5R上限的适配规则（widen-stop，仅适用于 TP1）：**
- 尖峰的测量运动目标（100%/138.2%/161.8%投影）作为 **TP2** 候选；**TP1** 仍须在 1.0–1.5R 内。
- 默认止损仍为最近回撤/趋势棒极点外1跳；当结构目标会导致回报÷风险 >1.5时，才允许在尖峰场景放大止损。
- 适配方法：把止损放远到满足 `结构目标 ≤ 1.5 × 止损距离`，但放大后止损不得超过尖峰高度50%。
  - 例：结构目标=30点，1.5R上限→止损至少20点；若20点 > 尖峰高度50%，则 reject。
- 尖峰起点止损只是不超过50%上限时的候选放大位，不是默认止损。
- 若放大后的止损 ≤ 尖峰高度50%且通过交易者方程 → 允许下单。
- 若放大后的止损 > 尖峰高度50% → 放弃，不缩小结构目标、不强行入场。
- 此规则**仅适用于尖峰场景**，普通通道/区间不得用 widen-stop；只能取1.5R内最近结构位或 reject。

**紧凑通道（tight_channel）专项规则：**
- 紧凑通道是强趋势，止损通常在1–2 ATR内。
- 在 §10.3 评估时，**首选目标不超过 1.5R**（回报 ≤ 1.5×风险），选通道内最近支撑/阻力或前波段测量位。
- 若 1.5R 内无结构支撑，可降至 1R；若方程仍不通过则 reject，并在 watch_points 说明触发条件。

## 11. 下单方式选择

### 11.1 是趋势 / 尖峰状态吗？

是：
- 优先止损单或市价单。
- 方向必须顺势。
- 尖峰追单必须更保守：trade_confidence 下调，且必须满足更严格的结构与跟随条件。

否：
- 继续。

### 11.2 是通道回撤吗？

是：
- 窄通道可用限价单在前一棒极点附近入场。
- 常规通道用止损单等待信号确认。
- 宽通道可用限价或止损，但必须接近边界。

否：
- 继续。

### 11.3 是交易区间吗？

是：
- 新手用止损单等确认。
- 老手可在边界用限价单。
- 中间位置不挂单。

否：
- 继续。

### 11.4 是突破交易吗？

是：
- 区间突破不追，除非尖峰级别。
- 最佳方式是突破后回踩测试入场。
- 若使用突破单，entry_price 必须挂在依据K线极点外：
  - 做多突破单：挂在信号棒或有效突破依据K线高点上方1跳动。
  - 做空突破单：挂在信号棒或有效突破依据K线低点下方1跳动。
- 突破单不得挂在K线实体中部、影线中部、EMA附近、当前价附近或任意折中位置。
- 若无法明确依据K线、高低点或最小跳动单位，**不要直接放弃**：评估是否在结构支撑/阻力/通道边界/EMA 处挂**限价单**；若 §10.3 方程可通过，可输出限价单方案。

否：
- 使用当前结构对应的入场方式（含限价单备选）。

## 12–13. 持仓后管理（PA Agent 不适用）

本程序仅输出**单次**阶段二 JSON（是否给一笔入场方案），不跟踪开仓后 K 线，不做任何持仓后管理（减仓/加仓/移止损/跟踪止损/状态切换调仓等）。

- **禁止**在 decision / watch_points 中写分批出场、移止损到盈亏平衡、持仓后逐 K 管理等指令。
- 若需表达「结构若失效则方案作废」，只写 `invalidation_condition` 与 `watch_points`（针对**尚未成交或重新分析**的条件），而非持仓操作。

## 14. 禁止行为清单

以下行为出现任一项，直接停止交易或放弃当前信号：

- 未识别周期位置就交易。
- 数据不足仍然下单。
- 在极端交易区间中交易。
- 在尖峰中做反转。
- 在买进高潮后追多。
- 在卖出高潮后追空。
- 在交易区间中追普通突破。
- 在区间中部入场。
- 在趋势型交易区间中做趋势跟踪。
- 在宽通道中追突破。
- 在常规通道中频繁逆势或在通道中部逆势交易（逆势仅限边界强信号）。
- 在趋势未双确认前做反转。
- 在 Always In Long 中做首个做空反转信号。
- 在 Always In Short 中做首个做多反转信号。
- 用当前输入周期长外包棒两端突破直接入场。
- 二次入场价格明显优于第一次仍强行交易。
- 二次入场失败后继续尝试第三次，除非是清晰楔形三推。
- 信号棒过长仍强行入场。
- 止损距离过大仍强行入场。
- 交易者方程不通过仍下单。
- 为了“看起来更容易成交/更好看盈亏比”而扭曲结构止损（扩大止损或贴噪音）以凑方程。
- 止损后立即反手。
- 连续两笔止损后仍不重新评估市场。
