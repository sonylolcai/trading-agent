## 获取数据（基础功能）

### 操作步骤

1. 在控制栏或 Web 终端工作台选择 **数据来源**（东方财富 / AkShare / Tushare / TradingView / MT5）
2. 选择 **品种**（如 `XAUUSDm`、`600519`）、**周期**（如 15m、1h、1d）
3. 若数据来源为 TradingView，可选 **交易所**（OANDA / PEPPERSTONE / TVC 等；默认「自动」自动探测）
4. 点击 **「获取数据」** 按钮
5. 图表开始刷新，状态栏显示当前数据来源和品种信息；等待状态栏确认数据获取成功

### 数据来源说明

| 来源 | 说明 |
|------|------|
| **东方财富** | A 股默认回退数据源，通过东方财富公开接口读取行情；适合 Web 端和 A 股日常分析 |
| **AkShare** | A 股数据源，部分场景可配合 Baostock 补齐历史数据 |
| **Tushare** | A 股数据源，需在 `config/settings.json` 的 `tushare.token` 填写 Token |
| **MT5** | 需 MetaTrader 5 终端运行并登录；品种名需与 MT5「市场报价」完全一致（含后缀如 `m`）；支持全部 MT5 时间周期（1m ~ 1M） |
| **TradingView** | 通过 tvDatafeed 拉取数据；可匿名使用（无需账号）；现货黄金、外汇、A 股、港股、加密货币、指数等均支持 |

Web API 中可通过 `GET /api/data-sources` 获取当前可见数据源，通过 `GET /api/timeframes?source=...` 获取该数据源支持的周期。

### 刷新机制

- 点击「获取数据」后，程序以设定间隔（默认 **1 秒**）持续从数据源拉取最新 K 线
- 新数据到达后自动更新图表（蜡烛图 + EMA20 叠加线）
- Web 端点击 **Fetch Data** 会调用 `POST /api/market/fetch` 拉取并写入本地 K 线缓存；图表快照和滚动回测优先读取缓存
- 最新未收盘 K 线以**浅色空心蜡烛**显示，不参与 AI 分析
- **切换品种 / 周期后，刷新自动停止**，需重新点击「获取数据」才会恢复拉取

### TradingView 交易所自动探测

选择交易所为「（自动）」时，程序按以下优先级自动探测可用数据源：

1. **A 股**：先尝试上交所（SSE），再尝试深交所（SZSE）
2. **港股**：使用港交所（HKEX）
3. **加密货币**：先尝试 BINANCE，再依次尝试 BITSTAMP、COINBASE 等
4. **指数**：SPX → NDX → VIX 等使用预设交易所
5. **黄金 / 外汇**：按 OANDA → PEPPERSTONE → FOREXCOM → TVC 顺序探测

探测过程中状态栏实时显示当前尝试的交易所；首个返回数据的交易所将被采用。

### 连接失败处理

- **MT5 未连接**：点击「获取数据」后状态栏提示"数据源未连接，请先切换数据来源"
- **TradingView 不可达**：弹出连接失败对话框，可选择切换到 MT5 或重试
- **Tushare Token 缺失**：补充 `config/settings.json` 中的 `tushare.token` 后重试
- **品种名无效**：品种输入框下方显示**红色警告**，需检查品种名是否正确
- **网络波动**：刷新循环自动指数退避（0.5s → 1s → 2s → … 最大 10s），恢复后自动归零

### 切换数据来源

- 切换数据来源时，图表自动重置，旧数据源断开连接
- 切换品种 / 周期时，图表自动重置，刷新停止，需重新点击「获取数据」
- 切换期间若有分析正在进行，分析将被取消

### 注意事项

- MT5 品种名必须与「市场报价」中名称完全一致（含后缀如 `XAUUSDm`）；A 股通常使用 6 位代码（如 `600519`、`000001`）
- K 线数据不足时，图表缓冲区需要数秒填满，期间状态栏提示"数据不足"
- TradingView 匿名模式数据可能受限，部分品种需登录账号
- 刷新间隔过短可能触发 TradingView 限流，建议不低于 1000 ms
- TradingView 连通性在点击「获取数据」时按需检测，而非程序启动时检测
